# Реализация и тестбенч протокола APB

## Обзор

Данный проект содержит **RTL-реализацию** протокола **Advanced Peripheral Bus (APB)** и **верификационную среду**.

- **RTL-дизайн** (APB_master, APB_slave, APB_top) взят из открытого репозитория Karankumar Nevage.
- **Тестбенч написан** на **SystemVerilog с использованием UVM**.

APB используется для подключения относительно медленной периферии (регистры, UART, SPI, GPIO, таймеры и т.д.) к основной системной шине. Он характеризуется минимальным количеством сигналов, простотой реализации и низким энергопотреблением.

В проекте используются:

- **Ведущий (Master)** APB — инициирует все транзакции
- **Два ведомых (Slave)** APB — содержат память и отвечают на запросы
- **Верхний модуль (APB_top)** — объединяет master и slaves

Система поддерживает операции чтения и записи, обработку ошибок и выбор ведомого по адресу.

## Тестбенч

При создании тестбенча использовался UVM 1.2. Кроме стандартных компонентов использовались базовые классы tb/base_classes. Подробное описание uvm-окружения находятся в файле UVM_TESTBENCH.md в папке tb.

## Что такое APB?

APB — это **синхронный** протокол с чётко определёнными фазами. Каждая транзакция (чтение или запись) проходит через следующие этапы:

1. **IDLE** — шина свободна, ничего не происходит.
2. **SETUP** — master выставляет адрес (`paddr`), выбирает slave (`psel`), указывает тип операции (`pwrite`: 1 = запись, 0 = чтение) и передаёт данные для записи (`pwdata`). При этом `penable = 0`.
3. **ENABLE (ACCESS)** — master поднимает сигнал `penable = 1`. В этот момент выбранный ведомый должен выполнить требуемое действие (записать данные в свою память или прочитать их) и ответить сигналом `pready = 1`.

Когда `pready = 1`, транзакция считается завершённой. Если slave нужно дополнительное время (например, для сложной операции), он может держать `pready = 0` (это называется **wait states**). При ошибке (неверный адрес и т.д.) ведомый поднимает `pslverr = 1`.

## Как работает система (общий принцип)

1. Внешний модуль подаёт сигнал `transfer = 1` + команду (`read` или `write`) + адрес + данные (при записи).
2. Master переходит из IDLE → SETUP → ENABLE.
3. По 9-му биту адреса (`paddr[8]`) master выбирает slave 1 или slave 2.
4. Выбранный slave выполняет операцию и поднимает `pready`.
5. При чтении master принимает данные от slave и выводит их на `apb_read_data_out`.
6. При ошибке (например, недопустимый адрес) slave выставляет `pslverr = 1`.

## Описание файлов дизайна

### 1. APB_master.v — Ведущий модуль

**Основная функция:** управляет всей шиной и инициирует транзакции чтения/записи.

**Реализация через конечный автомат (FSM):**

- **IDLE** (`2'b00`) — ожидание сигнала `transfer`.
- **SETUP** (`2'b01`) — подготовка:
  - Выставляет адрес, данные (`pwdata`), `pwrite`.
  - Активирует `psel1` или `psel2` (по биту [8] адреса).
  - `penable = 0`.
- **ENABLE** (`2'b10`) — активная фаза:
  - Поднимает `penable = 1`.
  - Ждёт `pready = 1` от slave.
  - При получении `pready`:
    - Если это **чтение** — сохраняет `prdata` в выход `apb_read_data_out`.
    - Если `transfer` всё ещё активен — сразу переходит обратно в SETUP (back-to-back транзакции без паузы).
    - Иначе — возвращается в IDLE.

Мастер готов к работе с wait states (если слейв задержит `pready`).

### 2. APB_slave.v — Ведомый модуль

**Основная функция:** хранение данных и выполнение команд мастера.

**Особенности:**

- Внутренняя память: **256 байт** (`reg [7:0] memory [255:0]`).
- Слейв **активен только** когда `psel == 1` **и** `penable == 1` (то есть именно в фазе ENABLE).
- Всегда выставляет `pready = 1` сразу после получения запроса (zero wait states в этой реализации).
- **Обработка адреса:**
  - Если `paddr < 8'd256` — нормальная операция.
  - Иначе — выставляет `pslverr = 1` и `pready = 1`.

**Операции:**

- **Запись** (`pwrite == 1`): `memory[paddr] <= pwdata;`
- **Чтение** (`pwrite == 0`): `prdata <= memory[paddr];`

### 3. APB_top.v — Интеграционный модуль

**Основная функция:** соединяет master с двумя слейвами в единую систему.

**Что делает:**

- Инстанцирует один `apb_master` и два `apb_slave`.
- Соединяет все внутренние сигналы (`penable`, `pwrite`, `paddr`, `pwdata` и т.д.).
- Мультиплексирует `pready` и `prdata` от двух слейвов (работает через OR, потому что одновременно активен только один слейв).
- Выбор слейва полностью происходит внутри master по 9-му биту адреса:
  - `paddr[8] == 0` → Slave 1 (диапазон 0x000–0x0FF)
  - `paddr[8] == 1` → Slave 2 (диапазон 0x100–0x1FF)

## Описания сигналов

### Сигналы APB_master

| Имя сигнала         | Тип    | Ширина | Описание                                  |
|---------------------|--------|--------|-------------------------------------------|
| `presetn`           | Input  | 1      | Активный по низкому уровню сигнал сброса  |
| `pclk`              | Input  | 1      | Тактовый сигнал (100 МГц в тестбенче)     |
| `transfer`          | Input  | 1      | Инициирует транзакцию                     |
| `read`              | Input  | 1      | Разрешение чтения (1 = операция чтения)   |
| `write`             | Input  | 1      | Разрешение записи (1 = операция записи)   |
| `apb_write_paddr`   | Input  | 9      | Адрес записи для slave                    |
| `apb_write_data`    | Input  | 8      | Данные для записи в slave                 |
| `apb_read_paddr`    | Input  | 9      | Адрес чтения для slave                    |
| `pready`            | Input  | 1      | Сигнал готовности slave (1 = готов)       |
| `pslverr`           | Input  | 1      | Сигнал ошибки slave (1 = ошибка)          |
| `prdata`            | Input  | 8      | Данные от slave во время чтения           |
| `psel1`             | Output | 1      | Сигнал выбора slave 1                     |
| `psel2`             | Output | 1      | Сигнал выбора slave 2                     |
| `penable`           | Output | 1      | Сигнал разрешения текущей передачи        |
| `pwrite`            | Output | 1      | Сигнал записи (1 = запись, 0 = чтение)    |
| `paddr`             | Output | 9      | Адресный сигнал для slave                 |
| `pwdata`            | Output | 8      | Данные для slave во время записи          |
| `apb_read_data_out` | Output | 8      | Выходные данные master при чтении         |
